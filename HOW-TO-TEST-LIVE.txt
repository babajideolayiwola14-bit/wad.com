WEB CHAT - TESTING YOUR LIVE SITE (Cloudflare)
==============================================

You do NOT need to run anything on your laptop to test the live app.
Skip 1-First-Time-Setup.bat and 2-Start-Chat.bat unless you want to
test on your computer later.


HOW TO TEST LIVE (every time)
-----------------------------
1. Open your website in Chrome or Edge (your normal address, e.g.
   https://your-domain.com)

2. Register or log in

3. Try posting a message, search, and logout/login again

4. If something breaks, write down:
   - What you clicked
   - What you expected
   - What happened instead
   - The time it happened


WHERE SETTINGS LIVE (not in .env on your laptop)
------------------------------------------------
Your live site reads settings from your HOSTING dashboard
(where the Node app actually runs — often Render, Railway, a VPS, etc.).

Cloudflare is often the "front door" (fast, secure address).
The app still needs DATABASE_URL and secrets on the real server host.

In that host's dashboard, check:
  DATABASE_URL     = PostgreSQL connection
  SECRET_KEY       = long random secret
  ADMIN_USERNAME   = admin login name
  ADMIN_PASSWORD   = admin password
  APP_URL          = https://your-real-site-address
  CORS_ORIGIN      = https://your-real-site-address


PASSWORD RESET EMAILS
---------------------
If reset emails should work live, also set SMTP_* variables there
(and APP_URL must be your live https address).


WHEN YOU CHANGE CODE IN CURSOR
------------------------------
1. Save your changes
2. Deploy/push the way you usually do for Cloudflare
3. Wait until the deploy finishes
4. Test again on the live URL (hard refresh: Ctrl+F5)


OPTIONAL: TEST ON YOUR PC
-------------------------
Only use the .bat files if you want to try changes before going live.
Most non-programmers can ignore this and only test the live site.
